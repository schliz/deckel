## Codebase Patterns
- Go module: github.com/k4-bar/deckel
- Entry point: cmd/server/main.go
- Empty directories use .gitkeep files for git tracking
- Go version: 1.24 (from go.mod)
- Use Go 1.22+ method routing: mux.HandleFunc("GET /healthz", handler)
- When adding pgx, must `go get github.com/jackc/pgx/v5/pgxpool` (not just pgx/v5) to pull in puddle dependency
- Migrations: SQL files in `migrations/` dir, embedded via `migrations/embed.go` package, goose uses `database/sql` via pgx stdlib driver
- Go version upgraded to 1.25 (goose v3.27 requires it)
- Store pattern: DBTX interface (Exec/Query/QueryRow) abstracts pool vs tx; Store.DB() returns pool as DBTX
- Renderer: New(templateDir, isDev, funcMap) parses layouts+partials+components+pages; Page() renders layout or content block for HTMX; dev mode re-parses on every request
- CSS build: `npx tailwindcss -i static/css/input.css -o static/css/styles.css --minify` or `scripts/build-css.sh`
- Tailwind 3.x + DaisyUI 4.x via npm (devDependencies); standalone CLI doesn't support DaisyUI plugin
- Fragment() supports both page templates and named sub-templates (components/partials) via Lookup() fallback
- Store queries are package-level functions: `store.GetUserByEmail(ctx, dbtx, email)` not methods on Store
- ErrNotFound sentinel error in store package for "not found" cases (wrap with errors.Is)
- FOR UPDATE pattern: lock user row first, then query balance within WithTx
- Multi-fragment HTMX response: Fragment() + AppendOOB() + raw HTML for multiple OOB swaps
- Template helpers: `map` (key-value pairs → map[string]any), `add`/`sub` (int arithmetic), `withinMinutes` (time check), `seq` (0..n-1 slice)
- Use `map` helper to pass multiple values to sub-templates: `{{template "foo" (map "Key1" val1 "Key2" val2)}}`
