# Contributing to schliz/deckel

Thank you for considering a contribution to this project! To keep things
transparent and legally sound, please follow these guidelines.

## Contributor License Agreement

Before your first contribution can be accepted, you must agree to our
[Contributor License Agreement](docs/CLA.md).

### How it works

1. Open a Pull Request as usual.
2. The CLA Assistant bot will automatically check whether you have already signed the CLA.
3. If you haven't signed yet, the bot will post a comment with a link.
4. Click the link, read the CLA, and sign it with your GitHub account — this is a **one-time step**.
5. Once signed, the bot will update the PR status to passing, and your contribution can be reviewed.

### Why a CLA?

This project is released under the MIT License. The CLA ensures that:

- The project owner retains the ability to manage and relicense the project.
- Contributors are properly credited and retain the right to use their own contributions.
- The project remains legally sound across different jurisdictions.

## How to Contribute

### Reporting Bugs

- Use the GitHub Issue tracker.
- Include steps to reproduce, expected behavior, and actual behavior.

### Submitting Changes

1. Fork the repository.
2. Create a feature branch from `master` (`git checkout -b feature/my-feature`).
3. Make your changes with clear, descriptive commit messages.
4. Push to your fork and open a Pull Request against `main`.
5. Ensure the CLA check passes (see above).

### Commit Messages

Please use clear and descriptive commit messages. We use the
[Conventional Commits](https://www.conventionalcommits.org/) format:

```
feat: add new authentication flow
fix: resolve race condition in database pool
docs: update API documentation
```

## Code of Conduct

Please be respectful and constructive in all interactions. We reserve the right
to remove comments or block contributors who engage in abusive or disruptive
behavior.

## Questions?

If you have questions about the CLA or the contribution process, feel free to open an issue.
